# PH6-L3D-004 — Runtime safety para imagens dinâmicas na Home

## Objetivo
Evitar quebra de render na rota `/` quando `next/image` recebe URLs dinâmicas inválidas ou com host remoto não permitido.

## Implementação
- Configuração de `images.remotePatterns` em `next.config.ts` para os provedores remotos utilizados no projeto (Cloudinary, Google, GitHub e Unsplash).
- Adição de guarda em `HomeExperience` para sanitizar `imageUrl` (salões) e `avatarUrl` (reviews):
  - aceita caminhos locais (`/images/...`);
  - aceita apenas URLs `http/https` de hosts permitidos;
  - faz fallback para assets locais quando a URL for inválida, malformada ou não permitida.
- Inclusão de teste cobrindo cenário com URLs inválidas/não permitidas e validação do fallback local.

## Arquivos alterados
- `next.config.ts`
- `src/components/home-3d/HomeExperience.tsx`
- `src/__tests__/Home3DExperience.test.tsx`
- `docs/development/tasks/PHASE-06-3d-scroll-design-migration.md`

## Resultado
- Home mantém render estável com dados reais e degradê seguro de imagens dinâmicas.
